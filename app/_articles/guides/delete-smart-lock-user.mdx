## Quick Answer

**To delete user: Open app ‚Üí Users/Codes ‚Üí Select user ‚Üí Delete/Remove ‚Üí Confirm ‚Üí Wait 2 minutes for sync ‚Üí Verify code no longer works on keypad.** Always test deleted code to confirm removal worked. For security, delete immediately after guest leaves or employee departs.

**Time required:** 2-3 minutes (delete + sync + verify)

## When to Delete Users

### Mandatory Deletion

```
Delete IMMEDIATELY when:
‚òëÔ∏è Guest checked out (Airbnb, VRBO)
‚òëÔ∏è Employee terminated or quit
‚òëÔ∏è Contractor finished job
‚òëÔ∏è Relationship ended (ex-partner, ex-roommate)
‚òëÔ∏è Lost phone with app access
‚òëÔ∏è Suspected security breach
```

### Routine Cleanup

```
Delete quarterly:
‚òëÔ∏è Temporary codes past expiration (if not auto-deleted)

‚òëÔ∏è Unused codes (haven't been used in 90+ days)
‚òëÔ∏è Test codes (from troubleshooting)
```

**Security rule:** If in doubt, delete it. Easy to re-add if needed.

## Step-by-Step Deletion

### Method 1: Delete via App (Most Common)

```
‚ñ° Open smart lock app
‚ñ° Select your lock (if multiple)
‚ñ° Navigate to Users / Codes / Access
‚ñ° Find user to delete
  - Scroll through list
  - OR: Search by name

‚ñ° Select user
‚ñ° Tap "Delete" or "Remove" or trash icon
‚ñ° Confirm deletion
  - App may ask "Are you sure?"
  - Confirm: Yes

‚ñ° Wait 2-5 minutes for sync
  - WiFi locks: 30-60 seconds
  - Zigbee/Z-Wave: 1-3 minutes
  - App shows "Synced" or checkmark

‚ñ° Verify deletion (CRITICAL)
  - Go to keypad
  - Try deleted code
  - Should give error beep (not unlock)
  - If still works: Wait 2 more minutes, try again
```

**Time:** 2-3 minutes total

### Method 2: Delete from Lock Keypad (Some Models)

```
Some locks allow direct deletion without app:

‚ñ° Enter programming mode
  - Usually: Master code + special button
  - Varies by brand (check manual)

‚ñ° Select delete user function
  - Navigate menu on keypad
  - Enter user slot number to delete

‚ñ° Confirm deletion
  - Keypad beeps confirmation

‚ñ° Exit programming mode

‚ñ° Test deleted code
```

**Brands supporting this:**
- Schlage  - programming code method
- Kwikset  - master code + program button
- Some Yale models

**When to use:** App not available, emergency deletion needed

### Method 3: Delete All Users (Factory Reset)

```
Nuclear option - use only if:
- Need to delete everyone at once
- Lost app access entirely
- Security breach  - delete all, start fresh

WARNING: Erases ALL codes and settings

Process varies by brand (see manufacturer manual)

After reset:
‚ñ° Set new master code immediately
‚ñ° Re-add only current/needed users
‚ñ° Update your records
```

## Common Issues

### Issue: Code Still Works After Deletion

**Cause:** Sync hasn't completed or failed

**Fix:**
```
‚ñ° Wait 5 minutes total (initial + extra time)
‚ñ° Wake up lock (press any keypad button)
‚ñ° Send lock/unlock command from app (forces sync)
‚ñ° Check lock is online (must be connected to sync)
‚ñ° Try deleting again if still not synced
```

### Issue: Can't Find User to Delete

**Cause:** User in different section or mislabeled

**Fix:**
```
‚ñ° Check all sections: Users, Codes, Access, Guests
‚ñ° Look for unnamed codes (generic "User 1")
‚ñ° Sort by date added (find recent)
‚ñ° Check if in "Expired" or "Inactive" section
```

### Issue: "Cannot Delete" Error

**Cause:** Master code or admin trying to delete self

**Fix:**
```
‚ñ° Can't delete master code (by design)
‚ñ° Can't delete your own admin access (safety)
‚ñ° Have another admin delete you
‚ñ° OR: Factory reset to remove master
```

### Issue: Accidental Deletion

**Cause:** Deleted wrong user

**Fix:**
```
‚ñ° Re-add user immediately (same code if possible)
‚ñ° Check app for "Undo" option (some apps, rarely)
‚ñ° Communicate with user about new code
‚ñ° Document incident (learn from mistake)
```

## Bulk Deletion

### Delete Multiple Users at Once

**For large cleanups (Airbnb season end, office turnover):**

```
‚ñ° Make list of users to keep (short list easier)
‚ñ° Delete all others one by one
  - Some apps: Select multiple ‚Üí Delete
  - Most: Delete individually

‚ñ° Verify after batch deletion
  - Test a few deleted codes
  - Spot check (don't need to test all 20)

‚ñ° Document remaining users
  - Who has access
  - What code they have
  - When added
```

**Time estimate:** 1 minute per user (including verification)

## Security Best Practices

### Deletion Protocol

```
When employee leaves:
‚ñ° Delete code immediately (within 1 hour)
‚ñ° Document deletion (date, time, reason)
‚ñ° Notify security/admin team
‚ñ° Change master code if they knew it
‚ñ° Review access logs for suspicious activity

When guest checks out:
‚ñ° Delete code same day (within 4 hours)
‚ñ° Verify deletion before next guest arrives
‚ñ° Clear code slot for reuse

When relationship ends:
‚ñ° Delete all access (codes + app)
‚ñ° Change master code
‚ñ° Consider physical lock change (if contentious)
```

### Audit Trail

```
Monthly review:
‚òëÔ∏è List all active users
‚òëÔ∏è Verify each should still have access
‚òëÔ∏è Check last used date (remove unused 90+ days)
‚òëÔ∏è Confirm temporary codes have expiration
‚òëÔ∏è Document review date
```

**Apps with audit logs:**
- August: Full access history
- Yale: Event log with user name
- Schlage: Recent activity log
